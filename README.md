# OSS Media Plus

`OSS Media Plus` 是一个基于 `React + Vite + TypeScript` 构建的阿里云 OSS 媒体工作台，聚焦图片预览、音视频在线播放和视频在线播放。

## 已实现能力

- 本地保存 OSS 配置
- 连接阿里云 OSS 并加载媒体文件
- 自动识别图片、音频、视频类型
- 图片卡片预览与大图查看
- 图片、视频全屏查看与新窗口打开
- 音频在线播放
- 视频在线播放
- 自定义时间轴，支持拖拽进度条
- 媒体类型筛选
- 目录浏览、创建文件夹
- 媒体库按文件名搜索整个媒体根目录
- 媒体库名称升序/降序排序
- 媒体库“加载更多”分页查询
- 移动文件到指定目录
- 上传前选择目标目录，并可直接新建目录
- 删除文件时先移动到回收站
- 回收站内支持预览、恢复到原位置、彻底删除
- 回收站支持多选、全选当前可见结果、批量恢复、批量彻底删除
- 归档/冷归档/深度归档对象一键发起解冻（RestoreObject）

## OSS 配置说明

需要填写以下信息：

1. `Region`
2. `Bucket`
3. `AccessKeyId`
4. `AccessKeySecret`
5. `媒体根目录`（可选）
6. `回收站基础路径`（可选，默认 `recovery/`）

说明：

- `媒体根目录` 用于媒体库、上传、浏览等主工作区，例如 `media/`
- `回收站基础路径` 按 Bucket 根目录解析，例如 `recovery/`
- 当配置为 `media/` 和 `recovery/` 时，两者是同级目录，不是上下级关系

配置会保存在浏览器 `localStorage` 中。由于本项目是“浏览器直连 OSS”，请务必使用最小权限的 RAM 账号，避免使用主账号 AccessKey。

## 上传目录选择

上传页面支持先选择目标目录，再选择或拖拽文件上传。

- 可在上传界面直接浏览目录
- 可在上传界面直接新建目录
- 上传任务会显示目标目录，避免上传后再手动移动

## 媒体库检索

媒体库支持在整个媒体根目录范围内做文件名搜索。

- 搜索范围固定为整个媒体根目录，而不是当前目录
- 搜索词为空时，列表回到当前目录浏览模式
- 搜索结果支持名称升序、名称降序
- 搜索结果使用“加载更多”方式分页追加

说明：

- v1 搜索仅匹配文件名，不做路径全文搜索、拼音搜索或标签搜索
- 为保证 OSS 直连场景下的结果稳定性，当前只提供名称排序，不提供按时间或大小的全局排序

## 预览增强

图片和视频预览支持以下操作：

- 全屏查看
- 新窗口打开

音频和视频保留自定义播放控制；视频在全屏状态下也支持浏览器原生控件。

## CORS 注意事项

浏览器直连 OSS 时，Bucket 需要提前配置跨域规则，至少建议：

- 允许当前访问域名
- 允许 `GET`、`HEAD`、`POST`
- 允许常见请求头
- 暴露 `ETag`、`x-oss-request-id`（以及你希望在前端读取的其他头）

## 存储类型与生命周期

当对象的存储类型为以下任意一种时，无法直接通过 `GET` 读取，需要先解冻（Restore）：

- `Archive`（归档）
- `ColdArchive`（冷归档）
- `DeepColdArchive`（深度归档）

如果你为音频/视频设置了生命周期规则（例如：上传后 1 天转为深度归档），那么在媒体库里点击播放时，可能会遇到 403，此时需要走“解冻流程”。

解冻耗时与存储类型相关，深度归档通常需要更长时间；解冻完成后会进入“可读状态”，在有效期内可直接访问，过期后会再次冻结。

相关文档：

- https://www.alibabacloud.com/help/zh/oss/developer-reference/restore-objects-11

## 解冻流程（系统内）

当你选中一个归档/冷归档/深度归档对象时，右侧预览区会提示“需要解冻后才能访问”，可执行：

1. 点击“发起解冻”向 OSS 提交 RestoreObject 请求
2. 解冻完成后，重试预览/播放
3. 如果提示链接过期或仍然 403，可点击“刷新链接”重新生成签名 URL

说明：

- 本项目默认将签名 URL 过期时间设置为 7 天，避免“解冻耗时超过 1 小时导致链接过期”的问题

## 上传默认打标签（用于生命周期策略）

由于生命周期规则不方便基于文件后缀/类型做过滤，如果你的文件在同一目录里混放，可以在上传时给对象打标签，然后在 OSS 生命周期规则里基于标签筛选。

本项目在上传时默认写入 2 个对象标签（Object Tagging）：

- `omp-media`: `image` / `audio` / `video`
- `omp-archive`: `deep`（音频/视频）或 `none`（图片）

你可以在 OSS 控制台创建生命周期规则：当 `omp-archive=deep` 时，1 天后转为深度归档（或你希望的存储类型）。

## 回收站机制

项目内删除采用“软删除”：

1. 在媒体库执行删除时，对象不会立即彻底删除
2. 系统会先将对象移动到回收站基础路径
3. 回收站页面中可继续预览媒体
4. 可选择“恢复到原位置”或“彻底删除”
5. 支持多选后批量恢复或批量彻底删除

说明：

- 恢复时会按原始对象路径还原
- 如果原路径已存在同名对象，系统会阻止恢复覆盖
- 正常媒体库目录浏览会自动隐藏回收站目录，避免混入主工作区
- 批量“全选当前可见结果”仅作用于当前筛选后已加载并显示在页面中的条目
- 批量恢复遇到路径冲突时，会跳过冲突项并汇总失败原因，不会覆盖已有对象

## 权限建议（最小权限）

根据功能使用情况，通常需要（RAM Policy）：

- `oss:ListObjects`
- `oss:GetObject`
- `oss:HeadObject`
- `oss:RestoreObject`
- `oss:PutObject`（上传/创建文件夹）
- `oss:PutObjectTagging`（上传时写入对象标签）
- `oss:CopyObject`、`oss:DeleteObject`（移动文件）
